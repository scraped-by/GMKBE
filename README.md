# GMkBPE

GMkBPE is a C++/CUDA implementation for maximal $k$-biplex enumeration on bipartite graphs. The main implementation lives in this directory; baseline methods are placed separately under `baselines/`.

## Repository layout

- `src/main.cu`: main GPU-enabled entry point for enumeration.
- `include/`: CUDA/C++ headers for graph representation, partitioning, set operations, maximality checking, and utility code.
- `preprocess/graph_conver.cpp`: preprocessing tool used to convert raw graph inputs into the format consumed by the main executable.
- `data/src/tmp_load.h`: graph loading helpers for prepared graph files.
- `CMakeLists.txt`: top-level build configuration for `gmkbpe` and `graph_convert`.
- `build.sh`: helper script that bootstraps dependencies, downloads one KONECT dataset, preprocesses it, builds the project, and runs a sample command.

## Requirements

- CMake >= 3.25
- CUDA Toolkit
- C++20 compiler
- OpenMP
- `fmt`
- `nlohmann_json`
- Optional: TBB for preprocessing

The main project targets NVIDIA GPUs and is configured for CUDA architectures `80;86;90` in `CMakeLists.txt`.

## Build

### Option 1: plain CMake

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
```

This produces:

- `build/gmkbpe`
- `build/graph_convert`

### Option 2: helper script

```bash
./build.sh
```

The helper script is intended for Linux. It detects CUDA, prepares a local `vcpkg` environment if needed, downloads a sample KONECT archive, builds the preprocessing tool and the main executable, and runs a sample command.

## Preprocessing

Prepared graph files are required before running `gmkbpe`. The preprocessing utility is built as `graph_convert` from `preprocess/graph_conver.cpp`.

Example build and use:

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --target graph_convert -j
./build/graph_convert -i <raw-input-file> -o <prepared-graph-file>
```

The helper script also demonstrates this flow automatically after extracting a KONECT input file.

## Run

Example:

```bash
./build/gmkbpe -f <prepared-graph-file> -k 1 -r 1000000 -d 0
```

Common parameters:

- `-f`: prepared graph file path
- `-k`: value of `k`
- `-r`: result limit
- `-d`: CUDA device id

Additional parameters may be available in the executable; use the source in `src/main.cu` as the authoritative reference.

## Input and output

The main executable reads a prepared graph file generated by the preprocessing stage. At runtime, it reports graph information, selected GPU/device information, timing, and checkpoint-style result-count/time records.

## Notes

- The helper script and the main implementation currently assume a Linux-like environment.
- Large datasets may require substantial GPU memory and host memory.
- Baseline implementations are stored under `baselines/` and are not covered by this README.
